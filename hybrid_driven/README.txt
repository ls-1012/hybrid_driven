混合驱动核心思想：
数据驱动+关键字驱动
数据驱动实现不同数据的验证
关键字驱动实现将每一个步骤封装成一个函数，通过执行测试步骤实现业务逻辑的功能

关键字驱动的实现：
visit|http：www.baidu.com--------command='visit("http：www.baidu.com")'-------eval(command)
def visit(url):driver.get(url)
业务逻辑的代码转换成一个一个的关键字函数，然后封装函数的实现即可。

框架实现的功能：
1、conf各种配置目录，包括页面元素的配置信息、日志配置信息
2、log日志保存目录
3、截图文件保存目录
4、测试数据文件保存目录
5、Action包，对每个动作的封装
6、Util包，工具类包，实现对各种操作的封装，如，打印日志，解析配置，操作Excel，截图，生成日期，操作文件目录，ObjectMap（将获取到的定位元素方法和定位表达式进行统一的元素查找）
7、测试脚本，主程序，执行测试数据，保存执行结果
